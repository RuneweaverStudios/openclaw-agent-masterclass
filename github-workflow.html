⚡ Performance Tips

Best practices for creating fast, efficient visual explainers:

  • Keep code blocks concise — Only show essential SQL, not full file contents
  • Use collapsible sections — <details> for advanced content that not everyone needs
  • Limit to 3-5 main sections — Too many sections overwhelm readers
  • Provide copy buttons — One-click code copying improves UX
  • Match soulmd.xyz aesthetic — Consistent themes build trust
  • Use descriptive filenames — kebab-case, topic-focused
  • Test locally first — Check rendering before pushing to GitHub
